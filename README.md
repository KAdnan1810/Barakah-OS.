# Autonomous Execution Rules

Treat this as a long-running engineering project.

Do not stop after a few files.

Continue until the application is complete.

Whenever output limits are reached, continue from the last completed file while maintaining consistency.

Maintain a project task list internally and complete every item.

Never regenerate completed files unless improvements are necessary.

Ensure all code compiles successfully before considering the project complete.

At the end, verify:

* The application builds successfully.
* There are no TypeScript errors.
* Database migrations work.
* Authentication works.
* All routes function.
* Charts render correctly.
* Mobile responsiveness is complete.
* Documentation is complete.
* The application is ready for deployment to Vercel with Supabase.

The final deliverable should be a complete commercial-quality SaaS application with clean architecture, production-ready code, and a polished user experience.
